SQL COMMIT and ROLLBACK

In SQL, the COMMIT and ROLLBACK statements help manage transactions. A transaction ensures data integrity by grouping one or more SQL statements into a single logical unit of work.

What is a Transaction?

A transaction is a group of operations executed together. If all succeed, the transaction is committed; if there’s an error or manual interruption, the transaction can be rolled back.

  • COMMIT: Permanently saves changes made by the transaction.
  • ROLLBACK: Reverts all changes made during the transaction.

COMMIT Syntax

COMMIT;

Use the COMMIT command to save all changes made in the current transaction.

ROLLBACK Syntax

ROLLBACK;

Use the ROLLBACK command to undo the current transaction before it's committed.

Example Using COMMIT

Here’s how you insert a record and commit the transaction:

START TRANSACTION; INSERT INTO Employees (ID, Name, Department) VALUES (5, 'Alice', 'IT'); COMMIT;

This ensures the new employee record is permanently saved.

Example Using ROLLBACK

If you detect a mistake during a transaction, you can cancel the changes:

START TRANSACTION; UPDATE Employees SET Department = 'Finance' WHERE ID = 5; ROLLBACK;

This rolls back the department update, discarding the change.

Important Notes

  • Some databases (e.g., MySQL) auto-commit each statement. Use START TRANSACTION or disable autocommit to control this behavior.
  • Transactions are ideal when multiple operations must either all succeed or all fail.
  • COMMIT makes changes permanent; ROLLBACK cancels them.
