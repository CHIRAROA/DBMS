PL/SQL Operators Precedence

In PL/SQL, operator precedence determines the order in which operators are evaluated in an expression. Operators with higher precedence are evaluated before operators with lower precedence. When two operators have the same precedence, their associativity (left-to-right or right-to-left) determines the order of evaluation.

Operator Precedence Table

Precedence Operator(s)
1 (Highest) **
2 + (Unary), - (Unary)
3 *, /
4 +, -
5 || (Concatenation)
6 RELATIONAL OPERATORS (=, <, >, <=, >=, <>, !=, ^=, IS NULL, LIKE, IN, BETWEEN)
7 NOT
8 AND
9 OR
10 (Lowest) :=

Example

DECLARE
    a NUMBER := 10;
    b NUMBER := 5;
    c NUMBER;
    result BOOLEAN;
BEGIN
    c := a + b * 2;
    DBMS_OUTPUT.PUT_LINE('Value of c: ' || c);

    result := (a > b) AND (c < 25);
    DBMS_OUTPUT.PUT_LINE('Logical Result: ' || result);
END;

Explanation

  • In the expression c := a + b * 2, multiplication has higher precedence than addition, so b * 2 is evaluated first, then added to a.
  • The logical expression (a > b) AND (c < 25) uses relational operators and a logical AND. Relational operators are evaluated first, followed by AND.

Notes

  • Use parentheses to explicitly define the order of evaluation when needed.
  • Misunderstanding operator precedence can lead to logic errors in your code.
