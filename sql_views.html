SQL CREATE VIEW Statement

In SQL, a view is a virtual table based on the result of an SQL query. It contains rows and columns, just like a real table, and the columns come from one or more actual tables.

You can use SQL statements and functions in a view to present data as if it were from a single table. A view is created using the CREATE VIEW statement.

CREATE VIEW Syntax

CREATE VIEW view_name AS SELECT column1, column2, ... FROM table_name WHERE condition;

Note: Views always display the most up-to-date data because the database engine recreates the result set every time the view is queried.

SQL CREATE VIEW Examples

Example 1: View of Customers from Brazil

Create a view showing all customers from Brazil:

CREATE VIEW [Brazil Customers] AS SELECT CustomerName, ContactName FROM Customers WHERE Country = 'Brazil';

Querying this view:

SELECT * FROM [Brazil Customers];

Example 2: View of Products Above Average Price

Create a view of all products with prices above the average:

CREATE VIEW [Products Above Average Price] AS SELECT ProductName, Price FROM Products WHERE Price > (SELECT AVG(Price) FROM Products);

Query the view:

SELECT * FROM [Products Above Average Price];

Updating a View

Use the CREATE OR REPLACE VIEW statement to update a view.

CREATE OR REPLACE VIEW Syntax

CREATE OR REPLACE VIEW view_name AS SELECT column1, column2, ... FROM table_name WHERE condition;

Example: Add "City" Column to Brazil Customers View

CREATE OR REPLACE VIEW [Brazil Customers] AS SELECT CustomerName, ContactName, City FROM Customers WHERE Country = 'Brazil';

Dropping a View

Use the DROP VIEW statement to delete a view.

DROP VIEW Syntax

DROP VIEW view_name;
