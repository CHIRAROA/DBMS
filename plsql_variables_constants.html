PL/SQL Variables and Constants

In PL/SQL, variables and constants are used to store data temporarily for use during program execution. A variable can be updated during execution, while a constant cannot be changed once assigned.

Declaring Variables

PL/SQL variables must be declared in the DECLARE section of a PL/SQL block. You can optionally initialize variables at the time of declaration.

Syntax

variable_name datatype [NOT NULL] [:= initial_value];

Example

DECLARE
salary NUMBER(7,2);
name VARCHAR2(50) := 'John';
BEGIN
NULL;
END;

Variable Scope

The scope of a variable is the region of the program where the variable can be accessed. PL/SQL supports:

  • Local Scope – Variables declared within a block, accessible only inside that block.
  • Global Scope – Variables declared in a package, accessible throughout the application.

Declaring Constants

Constants are similar to variables but must be assigned a value at the time of declaration and cannot be modified later.

Syntax

constant_name CONSTANT datatype := value;

Example

DECLARE
pi CONSTANT NUMBER := 3.14159;
app_name CONSTANT VARCHAR2(20) := 'HR_APP';
BEGIN
NULL;
END;

Using %TYPE Attribute

The %TYPE attribute allows you to declare a variable with the same data type as a column or another variable.

Syntax

variable_name table_name.column_name%TYPE;

Example

DECLARE
emp_salary employees.salary%TYPE;
BEGIN
NULL;
END;

Using %ROWTYPE Attribute

The %ROWTYPE attribute allows declaration of a record with the same structure as a row in a table or a cursor.

Syntax

record_name table_name%ROWTYPE;

Example

DECLARE
emp_rec employees%ROWTYPE;
BEGIN
NULL;
END;

Summary

PL/SQL supports robust variable and constant handling with options for type safety using %TYPE and %ROWTYPE. These constructs help in writing maintainable and error-free code, especially when working with database objects.
