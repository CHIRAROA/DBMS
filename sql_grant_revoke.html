SQL GRANT and REVOKE Statements

In SQL, managing user permissions is essential for maintaining database security. The GRANT and REVOKE statements are used to assign or remove privileges from users or roles.

GRANT Statement

The GRANT statement provides specific privileges to a user or role. These privileges define what operations they can perform on database objects.

Syntax

GRANT privilege_name(s) ON object_name TO {user_name | PUBLIC | role_name} [WITH GRANT OPTION];
  • privilege_name(s): Privileges to grant (e.g., SELECT, INSERT).
  • object_name: The target object of the privilege (e.g., table).
  • user_name: The user receiving the privileges.
  • role_name: A role to receive the privileges.
  • PUBLIC: Grants privileges to all users.
  • WITH GRANT OPTION: Allows the recipient to grant these privileges to others.

Example

Granting SELECT, INSERT, and DELETE privileges on the mydb database to user1:

GRANT SELECT, INSERT, DELETE ON mydb TO 'user1'@'localhost';

REVOKE Statement

The REVOKE statement removes specific privileges from a user or role, restricting what they can do in the database.

Syntax

REVOKE privilege_name(s) ON object_name FROM {user_name | PUBLIC | role_name};
  • privilege_name(s): The privileges to remove.
  • object_name: The object from which privileges are revoked.
  • user_name: The user losing the privileges.
  • role_name: The role losing the privileges.
  • PUBLIC: Revokes from all users.

Example

Revoking the INSERT privilege on mydb from user1:

REVOKE INSERT ON mydb FROM 'user1'@'localhost';

Understanding Privileges

Privileges determine what actions users can perform on database objects. Common ones include:

  • SELECT: Read data from a table.
  • INSERT: Add new rows to a table.
  • UPDATE: Modify existing data.
  • DELETE: Remove data from a table.
  • EXECUTE: Run stored procedures/functions.

Proper privilege management helps maintain database security and ensures users have only the access they need.
